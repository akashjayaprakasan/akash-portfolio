About me

From curiosity to code

My path into engineering started with a simple question: how do systems actually think and respond? That curiosity pulled me toward Computer Science, where data structures, algorithms, and clean architecture became the language I use to solve real problems.

Artificial Intelligence pulled me in early — the idea that a model could learn patterns from data and turn them into something useful, like a chatbot that understands symptoms through voice, felt like a glimpse of where software is heading. That fascination became my AI Healthcare Chatbot project.

At the same time, I never wanted to be confined to one layer of the stack. I enjoy full-stack development because it lets me own a problem end-to-end — from a REST API in Flask or FastAPI to a polished interface in React or vanilla JavaScript. Backend development, in particular, draws me toward the engineering discipline of building systems that are reliable under the hood, not just pretty on the surface.

What keeps me going is the process of problem solving itself: breaking down something messy into a structure that works, testing it until it holds up, and learning the part I didn't know yet. My vision is to grow into a software engineer who contributes to products that genuinely help people, while never stopping the habit of learning.

PROJECT 01 / AI + VOICE

AI Healthcare Chatbot with Voice Recognition

PythonFlaskScikit-learn PandasSpeechRecognitionHTML/CSS/JS

An AI-powered healthcare chatbot that lets users describe symptoms by typing or speaking, and returns predictions through a trained machine learning model — served via a Flask REST API to a responsive web interface.

Symptom-based health query chatbot
Flask REST API connecting model to frontend
Voice input via SpeechRecognition + Web Speech API
Real-time symptom-based predictions
Hands-free chatbot interaction
Tested across 20+ symptom scenarios
Architecture

A Flask backend exposes REST endpoints that wrap a Scikit-learn classification model trained on symptom data. The frontend (HTML/CSS/JavaScript) sends user input — typed or spoken — to the API, which returns a predicted condition and confidence-style guidance rendered back into the chat UI.

Challenges

Reliably mapping free-form spoken symptom descriptions to the structured input the ML model expected, and keeping voice recognition responsive across browsers without blocking the chat interface.

Solutions

Built a normalization layer between the Web Speech API output and the model's expected feature set, and used asynchronous calls so voice capture and prediction never froze the UI.

Key Learnings

Hands-on experience connecting a trained ML model to a live REST API, and designing voice-driven UX that degrades gracefully to typing when speech input isn't available.

Open Source Developer Analytics & Portfolio Generator

A full-stack platform that analyzes any GitHub profile to surface developer analytics, then generates a downloadable portfolio from that data — deployed end-to-end across Render and Vercel.

Architecture

A React frontend (deployed on Vercel) talks to a FastAPI backend (deployed on Render), which calls the GitHub REST API to pull repository metadata, commit history, and language statistics, then persists saved profiles in MongoDB Atlas.

Challenges

Aggregating data across 10+ repositories per profile within GitHub API rate limits, and turning raw commit/language data into a meaningful "developer score."

Solutions

Batched and cached GitHub API calls in the FastAPI layer, and designed a weighted scoring formula across activity, language diversity, and repository quality signals.

Key Learnings

End-to-end deployment experience across two separate platforms (Render + Vercel), and practical skills in API rate-limit management and cloud-hosted MongoDB.
